# Parte 08: Descripciones de resumen de commit

En esta sección, aprenderás cómo usar GitHub Copilot para generar mensajes de commit y personalizarlos.

1. [] Abre la vista **Git Changes** en Visual Studio (**View -> Git Changes**).
1. [] Selecciona el botón del lápiz con emoji de destello para generar automáticamente un mensaje de commit usando Copilot.

   ![VS commit dialog with AI button](./images/8-commit.png)
   
1. [] Navega a **Tools -> Options -> GitHub -> Copilot -> Source Control Integration**.
1. [] Actualiza la configuración de personalización del mensaje de commit a: `Summarize in a few sentences and then highlight the top 5 changes with emoji and short descriptions`
1. [] Regresa a la vista **Git Changes** y genera un nuevo mensaje de commit.
1. [] Observa cómo el nuevo mensaje de commit refleja la personalización actualizada.

**Punto clave**: Copilot puede ayudarte a crear mensajes de commit significativos y bien estructurados, ahorrando tiempo y mejorando la colaboración.

## Revisiones de código con Copilot

1. [] Antes de enviar tus cambios, usa la función de **Code Review** para analizar tu código en busca de posibles mejoras y sugerencias.

   ![Code Review Assistance toggle](./images/8-code-review.png)

1. [] Revisa las sugerencias proporcionadas por Copilot y aplica los cambios necesarios si los hay.

**Punto clave**: La función de Code Review de Copilot puede ayudarte a identificar posibles problemas y mejorar la calidad del código antes del envío.
